I used touch to create an original file, then created a hard link and a symbolic (soft) link. Using ls -li, I observed that the original file and the hard link share the exact same inode number, while the symbolic link has a different inode. I used stat to view the original file's metadata. After deleting the original file with rm, the hard link remained intact and accessible, but the symbolic link became broken because it merely pointed to the deleted file's path.
